Creating Challenges

Introduction

Whenever you want to create a new challenge, you have to abide by some rules. Failing to do so, will crash the app on your side and you will not be able to create a new challenge

In order to make your life easier as a user, the app was designed to abstract the testing library used to assert the outputs. This is why you need to provide the app with 2 main functions:

  • true function

    • This function will be used as a ground truth function. It is considered absolutely correct and will be used to compare the results that the user got with the results that the true function of the author returns for each random test case.

  • random function

    • This function will be used to generate random test cases and it is absolutely required to be random. Any challenge that fails to provide a correct random_function will be rejected in the validation process.

Requirements

Here is a simple example of how a true function and a random function should look like.

Below you will find an example on a challenge that requests users to return the difference between two numbers:

# Import any library you might use, but do not expect the user to be able to use them as well. # Only import libraries you need to write your random function e.g: random import random def random_function(): random_tuples = [] for _ in range(70): # Generate a random floating-point number between 0 and 1. random_num1 = random.uniform(0, 1) random_num2 = random.uniform(0, 1) # Add the numbers that were generated in the tuple. random_tuples.append((random_num1, random_num2)) return random_tuples

Time Limits

Some challenges may have a soft time limit.
All challenges have a 5 seconds hard time limit.

Soft Time Limit

This time limit is the one imposed by the author of the challenge. It may be used to create performance - related challenges.

Hard Time Limit

The hard time limit is imposed on each and every challenge on CodeLighthouse. There are no challenges that would require a solution to run more than the hard time limit on CodeLighthouse. It is practically hard - coded into the testing frameworks.
The purpose of this time limit is to:

  • Restrict and prevent malicious users from overloading the servers.

  • Restrict and prevent bad / accidental solutions that may have infinite loops coded from overloading the servers.
