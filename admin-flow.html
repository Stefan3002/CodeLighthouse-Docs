Admin Flow

Verdicts on challenges

As an admin, it is your duty to give verdicts on pending challenges. You will only be able to see challenges that were not made private, this is to respect the users' privacy. The only user that can see private challenges is the developer of the application who has access to the database itself. Let's respect our users privacy shall we?

Types of verdicts

  • Approve

    • Approving a challenge will make it immediately public. Admins, please, make sure it complies with the code of conduct of CodeLighthouse.

  • Send back

    • Sending back a challenge will signal to the original author that there are some things that are not alright.

  • Deny

    • Denying a challenge is a powerful action: it will forever remove the challenge from the platform. No appeal right. Please, admins, use this one carefully. Only use it when the challenge is very broken to the point where it is better to start over.

Reports

Another one of the admin's duties is to moderate and solve reports submitted by users.

Comments Reports

Because the application is free to use, all sorts of users may use it and spread hate around in their comments. When a user reports a comment, the admins will be presented with the ability to delete the comment if they decide that it violates the Code of Conduct.
When a user submits a report for a comment they must specify why they deemed that comment as inappropriate. This is to help admins make a decision.

Challenges Reports

Users are presented with 3 types of challenge-related reports:

  • Broken

    • This type of report reefers to the fact that the challenge itself is not working as expected, so there may be a problem related to the way that the author wrote the necessary functions.

  • Difficulty

    • This points out that the challenge is either too simple, or too difficult.

  • Description

    • The description may be too difficult or convoluted to understand
