Solve Challenges

Here you will find out everything you need to know related to solving challenges.

Basic Requirements

Whenever you solve a challenge, no matter which one you will be required to write a user_function function that will always take in a / an:

array of tuples: inputs => e.g: [(0, 1), (4, 5)]
array of arrays: inputs => e.g: [[0, 1], [4, 5]]
array of arrays: inputs => e.g: [[0, 1], [4, 5]]

Below you can find an example on how to write a function that correctly solves a challenge that requires you to return the difference of 2 numbers:

def user_function(inputs) return inputs[0] - inputs[1]
def user_function(inputs) inputs[0] - inputs[1] end

Awarding Points

Every single time you solve a new challenge, you will be awarded a certain amount of points. With this score in mind, each user will be granted a rank attached to his profile. Possible ranks include:

  • Dark 5 -> 1

  • Light 1 -> 5


    There is no category in between light and dark.

The way you receive points after solving a challenge depends on the difficulty of the challenge. The algorithm itself is not publicly available yet, but you can find an overview below regarding the base points that you get from each difficulty.

Time Limits

Some challenges may have a soft time limit.
All challenges have a 5 seconds hard time limit.

Soft Time Limit

This time limit is the one imposed by the author of the challenge. It may be used to create performance - related challenges.

Hard Time Limit

The hard time limit is imposed on each and every challenge on CodeLighthouse. There are no challenges that would require a solution to run more than the hard time limit on CodeLighthouse. It is practically hard - coded into the testing frameworks.
The purpose of this time limit is to:

  • Restrict and prevent malicious users from overloading the servers.

  • Restrict and prevent bad / accidental solutions that may have infinite loops coded from overloading the servers.
